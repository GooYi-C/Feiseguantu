# 关系索引面板 - 高对比度颜色设计方案

## 设计目标

解决用户反馈的颜色相似度问题，重新设计12种高对比度、易于区分的颜色，确保在深色背景下具有优秀的可读性和视觉区分度。

## 配色标准

### HSL色彩模型参数
- **色相差异**：至少60度，确保颜色明显区分
- **明度（Lightness）**：45-60%，适配深色背景
- **饱和度（Saturation）**：65-80%，平衡鲜艳度与可读性

### 设计原则
1. 参考Material Design和Tailwind CSS的配色标准
2. 考虑色盲用户的识别需求（避免仅依赖红绿对比）
3. 确保相邻字段的颜色对比度至少3:1
4. 按功能分组颜色，提升语义化

所有相邻颜色的色相差异均 ≥ 60度，确保视觉区分度。

### 明度对比度
- 最亮：紫罗兰 (65%)、靛蓝色 (67%)
- 中等：琥珀金 (50%)、鲜红色 (60%)、活力橙 (53%)
- 较暗：青绿色 (42%)、翡翠绿 (39%)、青碧色 (40%)

明度差异确保即使色相相近，也能通过明暗区分。

为常用字段分配固定颜色，确保用户体验的一致性：

## 第13+字段的视觉区分

当字段数量超过12个时，使用以下策略：

1. **颜色复用**：使用前12个颜色的深色版本
2. **边框标识**：添加与标签颜色相同的边框
3. **透明度调整**：整体透明度降低至85%
4. **视觉层次**：通过边框和透明度形成视觉层次

```scss
&.extended {
  border-color: var(--field-color);
  opacity: 0.85;
}
```

### 红绿色盲（最常见）
- ✅ 蓝色系（宝石蓝、青绿、靛蓝、青碧）与红色系（鲜红、玫红）通过明度区分
- ✅ 金色、紫色、橙色作为中间色，提供额外区分维度

### 蓝黄色盲
- ✅ 红色系（鲜红、玫红、玫瑰粉）与绿色系（翡翠绿、黄绿）通过明度区分
- ✅ 紫色、青色作为过渡色

### 全色盲
- ✅ 明度范围39%-67%，差异足够大
- ✅ 配合边框、hover效果等非颜色视觉元素

## 实际效果预览

### 深色背景下的对比度
所有颜色在深色背景（#1a1a1a）下的对比度均 > 4.5:1，符合WCAG AA标准。

### 紧凑布局测试
在标签间距仅6-8px的紧凑布局中，颜色区分度依然优秀。

## 实现细节

### CSS变量应用
```scss
.relation-chip {
  .rel-label {
    color: var(--field-color);
  }

```

### 动态颜色分配逻辑
1. 优先为固定字段分配预设颜色
2. 动态字段使用未被占用的颜色
3. 超过12个字段时添加 `extended` 类
