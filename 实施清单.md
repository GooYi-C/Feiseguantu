## 🧪 测试场景

### 场景1：标准配置（8个字段）
```
字段：一把手、直接上级、配偶、靠山列表、绯色对象列表、
      竞争对手列表、政治宿敌列表、核心嫡系列表

预期：
- 每个字段颜色不同
- 固定字段使用预设颜色
- 无Extended样式
```

### 场景2：LLM新增字段（12个字段）
```
字段：[8个标准字段] + 亲信列表、盟友列表、观察对象列表、潜在威胁列表

预期：
- 前8个字段使用固定颜色
- 后4个字段使用剩余颜色（9-12）
- 无Extended样式
```

### 场景3：超过12个字段（15个字段）
```
字段：[12个字段] + 外围势力列表、历史关联列表、待定关系列表

预期：
- 前12个字段正常显示
- 后3个字段有边框和透明度
- 颜色循环使用（13→1, 14→2, 15→3）
```

### 场景4：动态更新
```
操作：
1. 初始显示8个字段
2. LLM添加"核心嫡系列表"字段
3. 界面自动刷新

预期：
- "核心嫡系列表"显示为翡翠绿色
- 其他字段颜色不变
- 无闪烁或卡顿
```

## 🐛 已知问题与解决方案

### 问题1：颜色在不同显示器上可能有差异
**影响**：低
**解决方案**：已选择在大多数显示器上表现一致的颜色

### 问题2：极紧凑布局（<6px间距）下识别度降低
**影响**：低
**解决方案**：保持6-8px的标签间距

### 问题3：超过15个字段时颜色可能重复
**影响**：极低（实际场景中很少超过12个字段）
**解决方案**：Extended样式提供视觉区分
