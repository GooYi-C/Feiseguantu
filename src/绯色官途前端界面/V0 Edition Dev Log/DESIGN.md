# 绯色官途 · 前端设计方案

## 1. 背景与目标

- 为 LLM 文字游戏《绯色官途》提供沉浸式、可交互的变量展示与编辑界面，核心关键词：**政纪肃正 × 绯色暧昧 × 权谋危机感**。
- 面向 SillyTavern 酒馆助手 iframe，使用 **Vue + Pinia + Vue Router + zod**，并通过 `getVariables / replaceVariables` 同步游戏状态，可按需接入 MVU 变量。

## 2. 信息架构与导航

| 路由             | 模块            | 关联变量                                 |
| ---------------- | --------------- | ---------------------------------------- |
| `/`              | 仪表盘          | 时空舆情、机遇/危机摘要、待办            |
| `/scene`         | 当前场景        | 当前场景、在场人物                       |
| `/profile`       | 个人档案        | 个人档案、任职履历、在手项目、表彰/处分  |
| `/characters`    | 人物库          | 人物库(动态)、关系索引                   |
| `/relations`     | 关系图谱/网络   | 关系索引 + 人物库标签过滤 + 关系网       |
| `/variables`     | 全量变量编辑    | 全部 Schema 字段（可折叠/分页/分段编辑） |
| `/romance`       | 绯色关系/履历   | 人物库.绯色关系、绯色履历                |
| `/faction`       | 派系图谱        | 派系图谱                                 |
| `/assets`        | 个人资产        | 资产子表（房产/座驾/白手套）             |
| `/secrets`       | 暗账            | 把柄/地雷/人情债                         |
| `/opportunities` | 机遇与危机/待办 | 机遇与危机.当前机遇/潜在危机/待办        |

导航形态：顶部标题 + 汉堡菜单（窄屏），左侧抽屉导航（宽屏），面包屑保持层级。全站无横向滚动，主列自适应宽度 + `aspect-ratio` 控制卡片高度。人物总量在导航/关系网显著显示，快速得知人物库规模。

## 3. 模块蓝图

- 仪表盘：时间与地点卡、政治气候徽章、当日议题摘要；风险/机遇计分卡；待办列表；进度条（专案/清查）；时间轴展示“今日关键事件”。
- 当前场景：场景速写（可折叠全文）、在场人物标签墙、潜在议题卡。支持“标记处理/忽略”简易操作。
- 人物库：列表+筛选（派系、状态、关系标签、危险度/好感/信任区间）；搜索框；人物行展开基础信息；详情抽屉分区（官场关系/绯色关系/竞争/靠山/家庭/子女）。支持增删改、复制人物模板。
- 关系索引/图谱：核心角色快捷跳转；靠山/竞争对手/绯色对象列表；可选“分层矩阵视图”或“简化节点图”（小规模时启用）。
- 关系网（新增）：显示 {{user}} 与人物库所有角色节点，节点上方显示总人数；边颜色/样式区分关系类型（如靠山=金色实线、绯色=绯红渐变、竞争/宿敌=锯齿或虚线、家属=柔和灰蓝）。支持按“官场关系/绯色关系/竞争关系/靠山关系/家庭关系”一键过滤分组，左侧筛选区勾选后节点与边高亮/淡出。点击节点弹出角色卡抽屉（复用人物详情），支持从卡片跳转到绯色页。
- 绯色关系与履历：风险色阶、阶段进度条、情绪状态 Tag；敏感字段遮罩/解锁按钮；履历时间轴。
- 绯色互动页（强化）：独立页面突出绯色情绪与视觉；精美角色卡组件（头像/封面、关系阶段进度、危险度、标签墙、最近事件）；支持本地上传角色插图（见第 4 节数据策略），卡片内切换封面/头像、查看原始尺寸；支持“仅绯色对象”过滤。
- 全量变量编辑页（新增）：基于 Zod Schema 全字段渲染为“分区手风琴 + 子表格/列表 + 分页”形式，保证无遗漏。提供“搜索字段名/内容”“仅显示已填写”“展开/折叠全部”“分页/虚拟滚动”控制。每个字段支持一键进入编辑态，按类型渲染控件：文本输入、数字带钳制、枚举下拉、多选 Tag、0-100 滑条、动态 record 的行增删改、数组的行内编辑与顺序调换。编辑完成后可“保存此段/保存全部”，支持撤销未保存。
- 个人档案：基础信息卡、能力值雷达/条形对比、任职履历时间轴、在手项目栅格、表彰/处分列表。
- 派系图谱：我方派系卡 + 主要派系列表（实力评估色阶、关系标签）；可选“对比面板”显示友敌阵营。
- 资产与暗账：资产卡片（房产/座驾/白手套）、暗账列表（被握把柄/手握把柄/地雷/人情债）；风险等级以色阶和警示 Icon 呈现。
- 机遇与危机/待办：列表/看板切换；按紧急程度排序；关联人物跳转；截止时间倒计时/颜色提示。

## 4. 数据模型与状态管理

- 入口加载：`getVariables({ type: 'character', character_id: getCharacterId() })` 或相应类型；zod 校验（已在 ZOD SCHEME 提供）；Pinia store 按域拆分：
  - `useGlobalStore`: 时空舆情、当前场景。
  - `useCharactersStore`: 人物库、关系索引、绯色履历。
  - `useProfileStore`: 个人档案（基础/能力/任职/项目/奖惩）。
  - `useFactionStore`: 派系图谱。
  - `useAssetsStore`: 个人资产、暗账。
  - `useOpStore`: 机遇与危机、待办。
- 读写策略：
  - 读取：初次 load + 手动刷新按钮；可设置定时轻量轮询或事件触发（聊天/楼层变化时可选 `reloadOnChatChange`）。
  - 写入：表单编辑 → zod 预校验 → `replaceVariables`。对 record 字段使用 `klona` 去除代理后写回。
  - 脏检查：本地 `draft` 副本 + `isDirty` 对比，关闭抽屉/切页时提示未保存。
  - 合并策略：对动态 record 采用“后写覆盖 + 空值移除”规则；对列表型字段使用去重和排序。
- MVU 适配（可选）：如需监听消息楼层变量，`await waitGlobalInitialized('Mvu')` 后使用 `Mvu.getMvuData` / `replaceMvuData`，并解析 `display_data/delta_data` 做变量变化提示。
- 本地插图缓存（新增）：用户上传的角色插图不写入 stat_data，使用 `localforage`/indexedDB 或 `localStorage` 存 DataURL/Blob URL，键为角色 ID（姓名）。Pinia 中仅存储插图引用（本地 URL），避免同步到 LLM。提供“清除本地缓存”按钮。
- 全量变量编辑（新增）：所有字段的编辑操作最终仍走 zod 校验 + `replaceVariables`，可“按分区保存”降低写入体积；对动态 record（如人物库/子女/把柄等）统一封装增删改 helper，保证键名唯一与默认值补齐；分页/虚拟滚动时仅渲染可见项，确保性能。

## 5. 交互流与表单

- 录入/编辑：分段表单（基础、关系、风险、通联等），即时校验提示；敏感操作二次确认。
- 搜索/筛选：人物库与派系支持关键字 + 多选 Tag 过滤；支持“只看高危/机遇/待办”快捷筛选。
- 关系网交互：过滤开关（关系类型、危机/机遇、标签关键字）；节点点击展开抽屉；右上角显示当前节点数/已过滤数；支持拖拽平移、滚轮缩放、双击居中 {{user}}。
- 全量变量编辑交互：字段级“一键编辑”按钮 → 行内切换控件，Esc 取消、Enter 保存当前字段；分区有“保存本分区/撤销改动/展开全部/折叠全部”按钮；顶部“仅显示已填写”可隐藏默认空值；支持键盘 Tab 顺序遍历字段。
- 批量导入导出：提供“导出 JSON / 导入 JSON”入口，复用 `?raw` 文件读取或手动粘贴。
- 撤销/草稿：编辑未提交前存于本地状态，可一键“撤销改动”恢复服务端数据。

## 6. 组件与可视化建议

- 卡片：深色磨砂背景 + 金色描边（官方肃正系）；绯色互动区使用暗红渐变高亮。
- 图形：能力值雷达/条形、风险/机遇计分卡、时间轴、进度条。无额外库时用 CSS/原生 SVG；如需更丰富可后续引入轻量图表库。
- 模态/抽屉：人物详情与编辑在抽屉；敏感信息遮罩点击解锁。
- 手风琴：用于履历/项目/奖惩/把柄分组折叠。
- 关系网：采用 SVG 或 @pixi/react。节点尺寸随权重（靠山/宿敌略大），边样式按关系类型上色/虚线/锯齿；选中节点高亮其一阶邻居，其余降噪。
- 全量变量编辑：每分区为卡片+手风琴；列表/record 使用行内表格，提供“添加一行”“复制行”“删除行”按钮；枚举用下拉，0-100 用滑条+数值框联动，长文本用可展开 textarea，时间/日期字段提供格式提示与掩码。

## 7. 性能与 UX 保障

- 只在需要时渲染重型视图（人物详情懒加载）；长列表使用虚拟列表或分页。
- 避免频繁写回：表单提交/批量保存，再调用 `replaceVariables`。
- 统一日志：关键操作 `console.info`；错误 `console.error` + 用户提示（toastr）。
- 关系网性能：节点 < 300 走力导布局，>300 退化为分组列表 + 小型矩阵；缩放/拖拽节流。
- 全量变量性能：长列表/record 使用虚拟滚动或分页（如每页 20-50 行）；折叠默认关闭次要分区；仅在编辑或展开时渲染控件。

## 8. 开发里程碑（建议）

1) 搭建路由/Pinia/基础布局与主题变量；完成仪表盘最小可用。
2) 上线人物库 + 详情抽屉（读写打通）；关系网基础视图（人数提示、关系类型上色、节点抽屉）。
3) 补充派系/暗账/机遇危机视图与可视化；关系网高级过滤（按关系模块一键筛选）。
4) 绯色互动页强化（精美卡片 + 本地插图上传/缓存）；隐私模式、批量导入导出、定时刷新/事件刷新。
5) 全量变量编辑页：分区渲染、搜索/折叠/分页、类型化编辑控件、分区保存。
