# 绯色官途 · 架构重构方案

## 1. 现状分析

### 1.1 当前架构概览

```
src/绯色官途前端界面/
├── index.html          # 入口 HTML
├── index.ts            # Vue 应用初始化
├── app.vue             # 主布局组件
├── router.ts           # 路由配置 (11个路由)
├── stores/
│   ├── schema.ts       # Zod Schema 定义
│   └── game.ts         # 单一 Pinia Store
├── styles/
│   ├── variables.scss  # 主题变量
│   └── official-theme.scss
└── views/              # 11个页面组件
```

### 1.2 识别的核心问题

#### A. 架构层面问题

| 问题类型                   | 具体表现                                    | 影响程度 |
| -------------------------- | ------------------------------------------- | -------- |
| **Store 过度集中**         | `game.ts` 承担所有数据管理职责，约260行代码 | 🔴 高     |
| **组件耦合度高**           | Views 直接访问 gameStore，缺乏中间抽象层    | 🟡 中     |
| **缺乏统一的数据编辑机制** | Variables.vue 仅实现部分分区编辑            | 🔴 高     |
| **无类型安全的表单组件**   | 每个 view 重复实现表单逻辑                  | 🟡 中     |
| **事件监听不完善**         | 仅监听 `CHAT_CHANGED`，缺少消息楼层变化监听 | 🟡 中     |

#### B. 功能层面问题

| 功能缺失           | 当前状态                           | 优先级 |
| ------------------ | ---------------------------------- | ------ |
| 人物编辑功能       | `editCharacter` 仅显示 toastr.info | 🔴 P0   |
| 全量变量编辑       | 7个分区仅显示 JSON 预览            | 🔴 P0   |
| 新增人物表单       | 使用 `prompt()` 原生弹窗           | 🟡 P1   |
| 数据导入导出       | 未实现                             | 🟡 P1   |
| MVU delta 变化提示 | 未实现                             | 🟢 P2   |

#### C. UI/UX 层面问题

| 问题           | 表现                               | 影响     |
| -------------- | ---------------------------------- | -------- |
| 关系网固定尺寸 | `viewBox="0 0 800 600"` 不响应容器 | 布局问题 |
| 人物超多时性能 | 无虚拟滚动，>50人物时卡顿          | 性能     |
| 移动端适配不足 | 部分页面横向溢出                   | 可用性   |
| 脏数据提示位置 | 固定右下角，可能遮挡内容           | UX       |

## 2. 重构目标

### 2.1 核心目标

1. **完整的变量编辑能力** - 所有 Schema 字段可视化编辑
2. **模块化 Store 架构** - 按领域拆分，降低耦合
3. **可复用的表单组件** - 类型化控件，减少重复代码
4. **性能优化** - 虚拟滚动、懒加载、按需渲染

### 2.2 非目标（避免过度工程）

- 不引入复杂状态管理（如 Redux 模式）
- 不使用 Canvas/WebGL 重写关系网（SVG 已足够）
- 不实现实时协作功能

#### 核心 Store 示例

```typescript
// stores/useGameData.ts
export const useGameData = defineStore('gameData', () => {
  const rawData = ref<GameData>(GameSchema.parse({}));
  const loading = ref(false);
  const initialized = ref(false);
  const isDirty = ref(false);
  const dataSource = ref<'mvu' | 'character' | 'chat'>('character');

  // 数据加载策略
  async function loadData() {
    loading.value = true;
    try {
      // 优先 MVU -> 角色卡 -> 聊天
      const { data, source } = await loadFromBestSource();
      rawData.value = GameSchema.parse(data);
      dataSource.value = source;
      initialized.value = true;
      isDirty.value = false;
    } finally {
      loading.value = false;
    }
  }

  // 部分保存
  async function savePart<K extends keyof GameData>(key: K, value: GameData[K]) {
    const toSave = { ...klona(rawData.value), [key]: value };
    await saveToSource(toSave, dataSource.value);
    rawData.value[key] = value;
    isDirty.value = false;
  }

  return { rawData, loading, initialized, isDirty, loadData, savePart };
});
```

```typescript
// stores/useCharacters.ts
export const useCharacters = defineStore('characters', () => {
  const gameData = useGameData();

  const 人物库 = computed(() => gameData.rawData.人物库);
  const 人物列表 = computed(() => Object.entries(人物库.value));
  const 人物总数 = computed(() => 人物列表.value.length);

  // 按类型分组
  const 绯色对象 = computed(() =>
    人物列表.value.filter(([, p]) => p.绯色关系 || p.角色标签?.includes('绯色对象'))
  );
  // ...其他分组

  // CRUD 操作
  function addCharacter(name: string, data: Partial<人物>) {
    if (人物库.value[name]) throw new Error('人物已存在');
    gameData.rawData.人物库[name] = 人物Schema.parse({ 姓名: name, ...data });
    gameData.isDirty = true;
  }

  function updateCharacter(name: string, updates: Partial<人物>) {
    if (!人物库.value[name]) throw new Error('人物不存在');
    Object.assign(gameData.rawData.人物库[name], updates);
    gameData.isDirty = true;
  }

  function deleteCharacter(name: string) {
    delete gameData.rawData.人物库[name];
    gameData.isDirty = true;
  }

  return { 人物库, 人物列表, 人物总数, 绯色对象, addCharacter, updateCharacter, deleteCharacter };
});
```

#### FieldRenderer 组件设计

```vue
<!-- components/variable/FieldRenderer.vue -->
<template>
  <div class="field-row" :class="{ full: isFullWidth }">
    <label>{{ label }}</label>

    <!-- 文本 -->
    <input v-if="fieldType === 'string'" type="text" v-model="localValue" />

    <!-- 数字 -->
    <input v-else-if="fieldType === 'number'" type="number" v-model.number="localValue" :min="min" :max="max" />

    <!-- 百分比 (0-100) -->
    <SliderField v-else-if="fieldType === 'percentage'" v-model="localValue" />

    <!-- 枚举 -->
    <EnumSelect v-else-if="fieldType === 'enum'" v-model="localValue" :options="enumOptions" />

    <!-- 数组 -->
    <ArrayEditor v-else-if="fieldType === 'array'" v-model="localValue" />

    <!-- Record -->
    <RecordEditor v-else-if="fieldType === 'record'" v-model="localValue" :itemSchema="itemSchema" />

    <!-- 长文本 -->
    <textarea v-else-if="fieldType === 'textarea'" v-model="localValue" rows="3" />
  </div>
</template>

<script setup lang="ts">
import { computed } from 'vue';
import { inferFieldType } from '@/utils/schemaUtils';

const props = defineProps<{
  modelValue: unknown;
  schema: z.ZodTypeAny;
  label: string;
  path: string;
}>();

const emit = defineEmits(['update:modelValue']);

const localValue = computed({
  get: () => props.modelValue,
  set: v => emit('update:modelValue', v),
});

const { fieldType, min, max, enumOptions, itemSchema, isFullWidth } = inferFieldType(props.schema);
</script>
```

### 3.4 事件监听增强

```typescript
// stores/useGameData.ts
function setupEventListeners() {
  // 聊天变化时重载
  eventOn(tavern_events.CHAT_CHANGED, () => {
    console.info('[绯色官途] 聊天变化，重新加载');
    loadData();
  });

  // 消息楼层变化时检查 MVU 更新
  eventOn(tavern_events.MESSAGE_RECEIVED, async () => {
    if (dataSource.value === 'mvu') {
      await checkMvuDelta();
    }
  });

  // 消息编辑/删除时同步
  eventOn(tavern_events.MESSAGE_UPDATED, handleMessageUpdate);
  eventOn(tavern_events.MESSAGE_DELETED, handleMessageDelete);
}

async function checkMvuDelta() {
  const mvuData = Mvu.getMvuData({ type: 'message', message_id: 'latest' });
  if (mvuData?.delta_data && Object.keys(mvuData.delta_data).length > 0) {
    // 显示变化提示
    showDeltaNotification(mvuData.delta_data);
    // 可选：自动合并或提示用户
  }
}
```

```typescript
  dirtyFields.value.add(path);

  if (path) {
    dirtyFields.value.delete(path);
  } else {
    dirtyFields.value.clear();
```
